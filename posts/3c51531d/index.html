做题记录（nt错误）合集

2026-03-11

  • P1433 吃奶酪

    • 第一道状压（也许）！
    • 似乎是经典问题（Traveling Salesman Problem, TSP）简化版（
    • 搜索用普通剪枝过不了（这里 NNN 比较小，然后似乎感觉 DP 的时间复杂度确定一些？因为递归还要考虑栈）
    • 用 memset 的时候注意 memset 是按字节赋值的， double 赋值有bug

  • P1596 [USACO10OCT] Lake Counting S

    • 这是什么Flood Fill 问题来着，求连通分量个数
    • 然后我用 BFS 写入队没标记重复入队爆队列 RE 了（也没用循环队列）
    • 似乎 DFS 更直观？
    • BFS 结尾神秘 \n 导致 10pts WA

之前记在洛谷上的：

任何一个伟大的思想，都有一个微不足道的开始。

  • P5707 【深基2.例12】上学迟到
    这是自己不hack的问题，只用样例，测评数据倒是挺到位的

  • P5709 【深基2.习6】Apples Prologue / 苹果和虫子
    又是没hack（确实太懒了），总结一些hack点：边界条件（越界和卡界），除法取整问题。 。

  • P1980 [NOIP 2013 普及组] 计数问题
    float精度不够，不开double见祖宗。 。

  • P1914 小书童——凯撒密码
    ASCII码只有128个。 。

  • P1321 单词覆盖还原
    题解区做法好哇，覆盖并没有改变什么，只是变得更好输入了

  • P1518 [USACO2.4] 两只塔姆沃斯牛 The Tamworth Two
    做完看了看题解，发现有佬用农夫与牛的状态处理判断重复，很多用6维数组，最妙的一个是用农夫与牛的位置加权得到专属值，%%%

  • P1098 [NOIP 2007 提高组] 字符串的展开
    模拟题就该模拟，不要乱改写题意（比如反面的条件），一定要考虑边界情况！

  • P1786 帮贡排序
    当冒泡排序练习，两次排序多关键字，注意储存初始相对顺序

  • P14972 『GTOI - 2C』Fliping
    比赛题，错在n-2与n-1需要交换时没有特殊考虑，40pts

  • P1249 最大乘积
    对上界估计方面仍需严谨，当n较大误差也会很大

  • P1088 [NOIP 2004 普及组] 火星人
    似乎考虑 M≤100M\le100M≤100 就以为只要后六位全排列，还是能有数据hack掉的。考虑剪枝全排列dfs或者直接next_permutation唔。然后似乎有康托尔展开的背景（原来是火星人）

  • P1928 外星密码
    递归但是只能处理最先出现的一些括号套，还得遍历。考虑一个一个字符读入会比字符串方便一些（？），找数字至少不用分类讨论。

  • P1164 小A点菜
    似乎是01背包，实测dfs剪枝+记忆化才能AC

  • P3612 [USACO17JAN] Secret Cow Code S
    一个分治，呃，卡了我两天（awa，后面又是各种细节出错，且没开long long，基础不扎实……感觉这道题可能是从正面考虑，按照题中延长 sss 的顺序来要方便一些，我从 NNN 入手，结果就……了。看了题解发现就是这个位置好像本来就是字符串反转再平移了一下而已（

  • P1498 南蛮图腾
    要用数组！！！（卡了好久是因为想直接输出）看到有一个异或递推的（杨辉三角模2发现）的题解，感觉有点接近本质？

  • P1803 凌乱的yyy / 线段覆盖
    贪心。但是我是考虑优先选和其他比赛重叠最少的（可能不完全正确而且还TLE），而题解给出的做法普遍是优先选结束时间最早的（可以留更多时间去参加后面的比赛）。

  • P1106 删数问题
    一开始考虑删去比后一个数大的数这个贪心，然后发现这个删数有多轮，可能第一轮就把答案里的数删了。后来改进每轮只删一个数（我还不怎么用STL），这就等价于 k=1k=1k=1 的情形，又忘记前导 0 。最后改了前导 0 ，发现还有一个答案就是 0 的点。呃……
    然后看题解，有可以选 n−kn-kn−k 个数的，每次选第一位，设还要删 mmm 个数，那这个第一位必然在前 m+1m+1m+1 个数中产生，这样甚至可以边找边输出。orz 。然后，这似乎又是一个 RMQ 问题，有时间复杂度更优的算法，涉及一些数据结构之类的，之后学习一下，挖个坑（

  • P5019 [NOIP 2018 提高组] 铺设道路
    感觉很多贪心题都值得好好想一想，比如这个我感觉自己虽然一遍过了但是是一个递归的模拟，看了题解就有 O(n)O(n)O(n) 的做法，每次两个坑填下去， ans+=a[i]−a[i−1](a[i]>a[i−1])ans+=a[i]-a[i-1](a[i]>a[i-1])ans+=a[i]−a[i−1](a[i]>a[i−1])。因为题目只关心次数（结果），而不是其中的过程，那么并没有必要模拟，就是你一个一个坑填过去，然后如果后面的坑更浅或一样就顺便填了，如果更深还要花费次数再去填。

  • P4447 [AHOI2018初中组] 分组
    看到队列标签才想到用队列的（虽然只有用到及时出队降低时间复杂度），我是维护一个分组人数及实力最强者，然后更强的依次入队。题解区还有俄罗斯方块图形化（但确实很有道理）……有一个维护单调性并且使用二分查找的可能比我这个效率高（好像确实欸，一般人看到什么最少的最大就直接二分了）

  • P1080 [NOIP 2012 提高组] 国王游戏
    邻项交换法似乎不是 iii 和 i+1i+1i+1 项交换的意思

  • P3853 [TJOI2007] 路标设置
    二等分不一定最优（

  • P2895 [USACO08FEB] Meteor Shower S
    标记有无的桶就直接用bool吧，，，不然会 MLE

  • P1019 [NOIP 2000 提高组] 单词接龙（疑似错题）
    似乎s.length()返回无符号数，然后如果和有符号的-1比较，-1会被转化为有符号数
